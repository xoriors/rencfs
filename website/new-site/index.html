rencfs is an encrypted file system mounted with FUSE on Linux. Create encrypted directories and backup safely to untrusted servers.

Engineered for Privacy

Built on the principle that privacy should be performant and accessible. Rencfs leverages proven cryptographic primitives.

Audited Cryptography

Uses well-known, audited AEAD cryptography primitives. We rely on the `ring` crate for established security foundations.

Metadata Privacy

It's not just the content. We encrypt filenames, sizes, and directory structures to hide all actionable intelligence.

Memory Safety

Written in Rust to guarantee memory safety. Credentials are managed with `mlock` and `zeroize` to mitigate cold boot attacks.

High Performance

Optimized for concurrency with parallel writes and fast seeking. AES hardware acceleration is used where available.

Data Integrity

Write-Ahead Logging (WAL) ensures your data remains consistent even in the event of a crash or power loss.

Cloud Ready

Designed to be backed up to untrusted servers like Google Drive or Dropbox without leaking information.

How it Works

  • 1

    FUSE Interface

    Intercepts file system calls from the OS.

  • 2

    Encryption Layer

    Chunks data and encrypts using AES-256-GCM or ChaCha20Poly1305.

  • 3

    Structure Obfuscation

    Encrypts directory hierarchy and file metadata.

  • 4

    Storage

    Writes encrypted blobs to the underlying storage.
